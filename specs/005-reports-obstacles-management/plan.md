# Implementation Plan: Reports & Obstacles Management

**Branch**: `005-reports-obstacles-management` | **Date**: 2026-06-19 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/005-reports-obstacles-management/spec.md`

## Summary

Build the Reports & Obstacles Management module (`/reports`) for the On The Way Admin Dashboard. This replaces the current stub at `/reports` with a server-paginated, searchable, filterable reports list and a dedicated `/reports/:id` details page. The details page renders a full report description, an image gallery, an embedded Leaflet map pinned to the report's GPS coordinates, and a metadata sidebar with community votes and submitter reference. Admins can Approve, Mark as Urgent, Remove (with mandatory reason), and Flag User directly from the details page. All state is synced to URL search params; all data is fetched via React Query hooks backed by the shared Axios client with typed fixture fallbacks.

## Technical Context

**Language/Version**: TypeScript ~6.0 (strict mode)
**Primary Dependencies**: React 19, Vite, TanStack React Query v5, React Router v7, Axios, Shadcn UI, Radix UI, Tailwind CSS v4, Zod, React Hook Form, Lucide React, Leaflet + react-leaflet (already installed), Sonner (toasts)
**Storage**: N/A (read + moderation actions; no local storage)
**Target Platform**: Web browser (admin dashboard, desktop-first, 768px+)
**Project Type**: Web application (admin dashboard)
**Performance Goals**: Reports list interactive within 2s of navigation; search/filter results within 2s of input; skeleton within 200ms of page access; moderation action feedback within 3s of click
**Constraints**: Server-side pagination required; no duplicate API requests; no new runtime dependencies (Leaflet already present); map renders only when GPS coordinates are valid
**Scale/Scope**: Potentially hundreds to thousands of reports; all filtering and pagination delegated to backend

- **Data and State** ✅: Server data fetched exclusively via `useReports`, `useReportDetails`, `useApproveReport`, `useMarkUrgent`, `useRemoveReport`, `useFlag User` React Query hooks. URL search params are the sole truth for list filter/pagination state. Mutation hooks use `useMutation` + targeted `invalidateQueries`. No `useEffect` for fetching. Forms (removal reason) use React Hook Form + Zod.
- **UX Consistency** ✅: All components use Shadcn/Radix/Tailwind patterns. Skeleton loading cards, inline empty state, destructive Alert for errors, Dialog for remove confirmation (with Zod-validated reason dropdown), and toast feedback via Sonner for all moderation actions are explicitly planned. Keyboard navigation and accessible labels are required on all action buttons.
- **Performance** ✅: Server-side pagination enforced. `placeholderData: keepPreviousData` on reports list. Leaflet map renders conditionally on valid coordinates only. `staleTime` set to minimize redundant refetches. Route-level code splitting applicable to `ReportDetails` if bundle analysis warrants.
